# Judging criteria (edit me)

Use a simple rubric so teams know how they are scored.

| Area | Weight (example) | What judges look for |
|------|-----------------|----------------------|
| Contract quality | 25% | Clear binding vow, correct spaces, `max_steps` set sensibly |
| Task interest | 25% | Non-trivial but evaluable; good failure modes |
| Reproducibility | 25% | Deterministic reset with seed; published snapshot |
| Code & docs | 25% | Env server runs; README; honest metrics |

**Replace weights and rows** for your event.
